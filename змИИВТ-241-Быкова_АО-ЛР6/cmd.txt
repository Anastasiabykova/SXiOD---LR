«docker run -d -p 5432:5432 --name d_cont d_img:latest». - команда создания docker-контейнера с именем «d_cont» на основе образа «d_img:latest» 
•	флаг -d даст доступ к командной строке после создания контейнера;
•	5432 – это номер порта, на котором находится база данных;


«docker exec -it d_cont psql -d -U usr dbn». - команда запуска контейнера в интерактивной среде с интерпретатором СУБД postgres под пользователем с именем «usr»
Подробнее:
•	docker exec – подключение к работающему контейнеру.
•	флаг -it – для того чтобы работала интерактивная среда прямо в терминале
•	d_cont – название контейнера
•	psql – команда, которая говорит, что в нашей интерактивной среде (в командной строке) нужно запустить интерпретатор postgres
•	-U – флаг указания имени пользователя (user)
•	-d – флаг указания наименования базы данных.



«docker run -d -p 5432:5432 --name d_cont d_img:latest -v /d_data:/var/lib/postgresql/data». - команда создания контейнера с томом (volume)
•	-v /data:/var/lib/postgresql/data – это как раз место для volume
•	-/data – это папка, которая находится в нашей операционной системе; можно указывать любую папку на компьютере;
•	/var/lib/postgresql/data – это папка, которая находится внутри операционной системы контейнера; эта папка постоянная, она не меняется.


«docker build -t test_image:latest .» - команда создания контейнера из Dockerfile
•	флаг -t задает TAG (версию), у нас – latest (позднейшая);
•	точка в конце означает, что берём файл из текущей папки.
